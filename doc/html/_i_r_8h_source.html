24/* A few important comments.
25 IRInstr has no jump instructions.
26 cmp_* instructions behaves as an arithmetic two-operand instruction (add or mult),
27 returning a boolean value (as an int)
28
29 Assembly jumps are generated as follows:
30 BasicBlock::gen_asm() first calls IRInstr::gen_asm() on all its instructions, and then
31 if exit_true is a nullptr,
32 the epilogue is generated
33 else if exit_false is a nullptr,
34 an unconditional jmp to the exit_true branch is generated
35 else (we have two successors, hence a branch)
36 an instruction comparing the value of test_var_name to true is generated,
37 followed by a conditional branch to the exit_false branch,
38 followed by an unconditional branch to the exit_true branch
39 The attribute test_var_name itself is defined when converting
40 the if, while, etc of the AST to IR.
41
42 Possible optimization:
43 a cmp_* comparison instructions, if it is the last instruction of its block,
44 generates an actual assembly comparison
45 followed by a conditional jump to the exit_false branch
46*/
