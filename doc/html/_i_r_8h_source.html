25/* A few important comments.
26 IRInstr has no jump instructions.
27 cmp_* instructions behaves as an arithmetic two-operand instruction (add or
28 mult), returning a boolean value (as an int)
29
30 Assembly jumps are generated as follows:
31 BasicBlock::gen_asm() first calls IRInstr::gen_asm() on all its instructions,
32 and then if exit_true is a nullptr, the epilogue is generated else if
33 exit_false is a nullptr, an unconditional jmp to the exit_true branch is
34 generated else (we have two successors, hence a branch) an instruction
35 comparing the value of test_var_name to true is generated, followed by a
36 conditional branch to the exit_false branch, followed by an unconditional
37 branch to the exit_true branch The attribute test_var_name itself is defined
38 when converting the if, while, etc of the AST to IR.
39
40 Possible optimization:
41 a cmp_* comparison instructions, if it is the last instruction of its block,
42 generates an actual assembly comparison
43 followed by a conditional jump to the exit_false branch
44*/
